# CLAUDE.md — ms-knowledge-base

## Project Overview

Local MCP server providing semantic search over curated Microsoft technical content (Fabric, Data Engineering, AI Engineering, Purview). Runs on a Beelink mini PC (Intel N100, 16GB RAM, Windows 11).

## Running Tests

```bash
python -m pytest tests/ -v
```

All tests use `tmp_path` fixtures for database isolation. Integration tests (`test_integration.py`) load the real embedding model and take ~30-50s.

## Key Commands

```bash
# Ingest a PDF
python scripts/ingest.py --file content/microsoft-learn/fabric-fundamentals.pdf.pdf

# Ingest a directory
python scripts/ingest.py --dir content/

# Show database stats
python scripts/ingest.py --stats

# Start MCP server (stdio for Claude Desktop)
python -m ms_knowledge_base.server.main --transport stdio

# Start MCP server (SSE for remote/network access)
python -m ms_knowledge_base.server.main --transport sse --port 3200
```

## Architecture

```
src/ms_knowledge_base/
  db/         — SQLite + sqlite-vec + FTS5 schema and CRUD
  ingest/     — PDF reader, heading-aware chunker, embedder, pipeline
  server/     — MCP server (FastMCP), hybrid search engine, auth
config/       — settings.py with all constants and topic keywords
scripts/      — CLI entry points (ingest.py, serve.py)
tests/        — pytest suite (unit, search, integration)
data/         — knowledge.db (gitignored, generated by ingestion)
content/      — source PDFs organized by type (gitignored)
```

## Code Conventions

- All dataclasses use `slots=True`
- Type hints on all functions (use `str | None` style, not `Optional`)
- Embedding model loaded ONCE at startup, passed through to components
- SQLite WAL mode on every connection
- sqlite-vec virtual table for vector search, FTS5 for keyword search
- Heading breadcrumb prepended to chunk content for embedding context
- No GPU code — CPU only (`device="cpu"`)
- No external API calls at query time

## Database

- `data/knowledge.db` — SQLite with WAL mode
- Tables: `sources`, `chunks`, `chunk_embeddings` (vec0), `chunks_fts` (fts5)
- FTS5 sync triggers on `chunks` table keep `chunks_fts` up to date
- Must delete from `chunk_embeddings` before `chunks` (virtual tables don't cascade)

## Topic Classification

Topics are assigned by keyword matching from `config/settings.py:TOPIC_KEYWORDS`. Taxonomy covers `fabric/*`, `purview/*`, `data-engineering/*`, `ai-engineering/*`, and `patterns/*`.
